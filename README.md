## ‚öôÔ∏è **Setup Process**

### 1Ô∏è‚É£ Create the Project Layout

The following high-level layout was created to keep application logic, utilities, and front-end assets clearly separated:

```bash
mkdir -p app/utils
mkdir -p static templates .circleci
touch app/__init__.py app/utils/__init__.py
```

* `app/` will host the main web application (for example, the future `app.py` entrypoint and route handlers).
* `app/utils/` will hold reusable helper functions and classes shared across the app.
* `static/` and `templates/` are reserved for the HTML/CSS/JS front end.

### 2Ô∏è‚É£ Enable Module Imports with `setup.py`

A minimal `setup.py` was added so that the project can be installed in **editable** mode.
This allows you to import `app` and `app.utils` from anywhere once the environment is activated:

```bash
uv pip install --editable .
```

Using editable installs ensures that changes to the source code are immediately reflected without reinstalling the package.

### 3Ô∏è‚É£ Create and Activate the Virtual Environment (uv)

The project uses **uv** to manage the virtual environment and locked dependencies.

Create the environment (Python 3.12, for example):

```bash
uv venv --python 3.12
```

Activate it:

**Windows (cmd)**

```cmd
.\.venv\Scripts\activate
```

**Windows (PowerShell)**

```powershell
.\.venv\Scripts\Activate.ps1
```

**macOS / Linux**

```bash
source .venv/bin/activate
```

### 4Ô∏è‚É£ Install Dependencies

With the environment active, install the initial dependencies defined in `requirements.txt`:

```bash
uv pip install -r requirements.txt
uv lock
```

The `uv.lock` file pins exact versions, supporting reproducible environments across machines and CI.

### 5Ô∏è‚É£ Set Up Environment Variables (`.env`)

Create a `.env` file in the project root:

```bash
touch .env
```

For now, this file acts as a placeholder for future configuration (for example, API keys for LLM or vision providers).
As additional components are introduced (e.g., external model APIs, storage backends), the corresponding keys will be added here and loaded via your configuration layer.
